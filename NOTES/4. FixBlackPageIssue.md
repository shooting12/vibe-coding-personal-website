# 搶救 Vite + React GitHub Pages「一片空白」終極指南 🚑

當您開心把網站推上 GitHub Pages 卻發現打開網址「全白」而且 Console 跑出 `Failed to load resource: the server responded with a status of 404()` 時，這不是您的程式碼寫錯，而是以下兩個最經典的部署設定地雷：

---

## 💣 地雷一：資源路徑對不上 (Base Path Mismatch)

### 原因
Vite 預設會把打包的資源路徑寫成絕對路徑 `/` (例如 `<script src="/assets/index.js">`)。但在 GitHub Pages 上，如果您的網站是放到 `https://您.github.io/您的Repo名稱/` 這種子路徑下，瀏覽器會直接跑去根目錄 `https://您.github.io/assets/index.js` 尋找檔案，當然會完全找不到 (404)！
就算您設定了 Repo 名稱當作 base，只要**字母大小寫**稍微打錯，或者往後更改了 Repo 名稱，都會立刻產生相同的白畫面災難。

### 解決方案：改成無敵的相對路徑 `./`
打開您專案的 `vite.config.js`，將 `base` 統一設定為 `./`：
```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

// https://vite.dev/config/
export default defineConfig({
  plugins: [react()],
  base: './', // 從絕對路徑改為相對路徑，讓它能在任何資料夾深度正常活著
})
```
設定完後，記得一定要 `git commit` 並 `git push` 到 GitHub！

---

## 💣 地雷二：GitHub 伺服器送出的是原始碼，不是完成品

### 原因
如果您的瀏覽器 console 出現的是錯誤尋找 `src/main.jsx`，這代表著您的部署流程有問題！
預設的 GitHub Repo 如果沒有設定好，它會把您推上去的「開發用原始碼」直接當成網頁丟出來。因為 GitHub Pages 本身沒有開發伺服器，所以網頁遇到需要編譯的 `main.jsx` 就會瞬間陣亡跳 404。

### 解決方案：切換為 GitHub Actions 來源
您必須告訴 GitHub：「請不要拿我的原始碼出來見客，請拿我透過 GitHub Actions 打包好的成果！」

1. 進入您專案的 **Settings** (設定)。
2. 點擊左側 Navbar 的 **Pages**。
3. 找到 `Build and deployment` 區塊，**請務必把 Source 下拉式選單切換成 👉 `GitHub Actions`**。
   *(如果您設定成 Deploy from a branch，GitHub 就會直接輸出原始 HTML)*
4. 設定完成後，回到專案的 **Actions 標籤頁**，找到最新一次的失敗部署紀錄。
5. 點擊右上方的 **"Re-run all jobs"** 按鈕讓它重新打包發布。
6. 等待綠燈亮起後，回到網頁強制重新整理 (`Cmd + Shift + R`)，白畫面就會徹底消失了！
