# 如何將個人網站部署到 GitHub Pages

這份文件記錄了如何透過 GitHub Actions 將現有的 Vite + React 網站自動化部署到 GitHub Pages。
目前專案中已經預先設定好了自動部署檔案 (`.github/workflows/deploy.yml`)。

未來的部署只要依循以下 4 個步驟：

## 步驟 1：確認您的 GitHub Repository 名稱
首先，從您的 Github 帳號建立一個全新的 Repository。

**重要設定**：如果您的 Repo 名稱不是 `您的帳號.github.io` (例如：`Vibe-Coding-Personal-Website`)，請務必先在專案目錄的 `vite.config.js` 加上 `base` 路徑。

請打開 `vite.config.js` 並加入 `base` 設定：
```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  // 如果您的 Github Repo 名稱是 Vibe-Coding-Personal-Website，請加上這行，確保資源路徑正確：
  base: '/Vibe-Coding-Personal-Website/',
})
```
*(💡 備註：如果您的 Repo 名稱就是 `您的帳號.github.io`，則不需要加上 base 這一行)*

---

## 步驟 2：將程式碼推送 (Push) 到 GitHub
打開您的系統終端機 (Terminal)，確保路徑位於專案資料夾內，然後依序輸入以下指令建立並上傳：

```bash
# 1. 建立 Git 追蹤 (若尚未建立)
git init

# 2. 將所有檔案加入變更區域
git add .

# 3. 提交變更紀錄
git commit -m "🚀 Ready for deployment"

# 4. 指定主分支為 main
git branch -M main

# 5. 綁定 GitHub 上的遠端 Repository (請取代為您的網址)
git remote add origin https://github.com/您在Github的帳號名稱/您的Repo名稱.git

# 6. 推送上去
git push -u origin main
```

---

## 步驟 3：在 GitHub 上開啟 Pages 功能
1. 回到瀏覽器，打開您建立的 GitHub Repository 頁面。
2. 點擊上方的 **Settings**（設定）標籤。
3. 在左側選單往下滑，選擇 **Pages**。
4. 找到 `Build and deployment` 的區塊，在 **Source** 下拉式選單中，選擇 **GitHub Actions**。

---

## 步驟 4：等待部署完成 ✨
切換到 GitHub Repo 上方的 **Actions** 標籤頁。
您會看到一個名為 `Deploy personal website to GitHub Pages` 的流程正在執行中。大約等待 1 到 2 分鐘，如果一切順利亮起綠色勾勾（✅），您的個人網站就成功上線了！

上線網址通常會是：
`https://您在Github的帳號名稱.github.io/您的Repo名稱/`
