// meta.svg
在程式碼裡看到的這長長一串： <path d="M6.915 4.03c-1.968 0-3.683... (一大堆數字和英文字母)"/>

這其實是 **SVG（可縮放向量圖形）**的「畫筆指令」！

SVG 跟一般的圖片（像 JPG 或 PNG）不同，它不是由一顆一顆的像素（Pixel）拼起來的，而是由 「純粹的數學公式」 寫成的。這就是為什麼 SVG 放大一萬倍都還是極度銳利、不會有馬賽克的原因。

而那串長長的路徑之所以那麼複雜，原因有幾個：

1. Meta Logo (無限符號) 其實充滿複雜的曲線
一般簡單的形狀（比如正方形）路徑會很短，因為它只需要告訴電腦「畫四條直線」： <rect width="10" height="10" />

但 Meta 的標誌是一個連續扭曲的 3D 質感緞帶（無限符號 ∞）。為了讓這個符號在任何螢幕上看起來都「完美平滑沒有死角」，設計師必須用非常多個「貝茲曲線 (Bézier curves)」來計算弧度和轉折點。

2. 字母背後的指令密碼
如果您仔細看那串密碼，會發現它裡面參雜了特定字母：

M = Move (移動畫筆到某個起點)
C / c = Curve (畫一條精密的貝茲曲線)
L / l = Line (畫一條直線)
A / a = Arc (畫一個圓弧)
Z = Close (封閉這個圖形並填滿顏色)
所以這長長的一串，其實就是電腦的「繪圖說明書」。電腦讀懂這串指令後，不用 0.001 秒就能在螢幕上算出並畫出那個平滑完美的 Meta Logo！這也是為什麼我們把它直接貼在程式碼裡，遠比去載入一張傳統圖片還要快速且穩定。


「既然我們剛剛有用指令把 meta.svg 抓下來放在 src/assets/ 裡面，為什麼還要大費周章把它裡面的程式碼複製、貼進 Experience.jsx 呢？為什麼不像 Aurora 那樣直接用 <img src={meta.svg} /> 把它當成圖片讀進來就好？」

這就牽涉到 React 開發中一個非常實用的進階技巧：內聯 SVG（Inline SVG）。

我選擇把它「解壓縮」直接寫進程式碼，主要是為了這兩個強大的好處：

1. 可以直接用程式碼「改變它的顏色」
如果您把 SVG 當成 <img /> 照片讀進來，它就只是一張死板的圖片。如果原始檔案是黑色的，它就永遠是黑色的，您沒辦法輕易改變它。 但因為我們把 SVG 程式碼直接貼進了 React 元件，它就變成了網頁結構的一部分！ 所以您可以注意到我在那段程式碼裡加了這個屬性：fill="#0668E1" 這代表我可以直接用語法指定它變成「Meta 的官方原廠藍色」。如果哪天您網站切換成超級深色模式，我們甚至可以直接寫 fill="white" 讓它瞬間變成白色！

2. 減少一次網路請求 (HTTP Request)
當瀏覽器看到 <img src="..."> 時，它必須發送一個額外的請求去伺服器把這張圖片抓下來。雖然檔案很小，但把這麼短的 SVG 程式碼直接寫在元件裡，意味著不用等圖片載入，Logo 會跟著您的文字在同一瞬間、零延遲地顯示出來，完全不會有圖片載入那 0.1 秒的閃爍。

這就是網頁工程師在處理這類「純幾何小圖示（Icon）」時，最喜歡用的手法！至於像 Aurora 那種包含複雜圖案、甚至是真實照片的照片，我們就會乖乖用 <img /> 標籤來讀取檔案了。
