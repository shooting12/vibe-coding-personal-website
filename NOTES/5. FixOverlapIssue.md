# 什麼是 RWD？(響應式網頁設計)

RWD（Responsive Web Design）是網頁開發中最核心的觀念。概念是 **「只寫一套程式碼，就能自動適應各種大小的螢幕」**。透過 RWD，網頁會根據使用者的螢幕尺寸（手機、平板、桌機）自動變換排版方式。

以下用本專案實際修復手機版「NavBar 與 Hero 重疊問題」為例，拆解 RWD 的 3 大招式：

## 1. RWD 的靈魂核心：CSS `@media` (媒體查詢)
在 CSS 檔案裡面的 `@media (max-width: xxx px)` 就像是寫給瀏覽器的 **`if...else` 條件判斷式**。

以專案中 `NavBar.css` 裡的程式碼為例：
```css
/* 平常狀態（桌機）：把 Navbar 往上提，製造緊湊感 */
.nav-bar-container { margin-top: -3rem; }

/* 變身機制：當螢幕寬度「小於或等於 900px （手機/平板）」時，啟動以下覆蓋規則： */
@media (max-width: 900px) {
  .nav-bar-container { margin-top: 1rem; } 
  .nav-link { font-size: 0.85rem; } /* 字體改小，免得手機塞不下 */
}
```
這個條件就像一道隱形的門檻，當使用者的視窗縮到 900px 內，瀏覽器就會瞬間套用大括號裡的覆蓋規則。這就是為什麼剛剛我們一加回 `margin-top: 1rem`，重疊問題就解決了！

---

## 2. 排版方向的彈性轉換 (Flexbox 方向)
這是網站設計最常使用的經典技巧。
在電腦 27 吋大螢幕上，您的**大頭照**跟**自我介紹**可以舒服地「左邊照片、右邊文字」排排站不會覺得擁擠。但到了手機螢幕這點寬度絕對裝不下水平列。

因此在 `Hero.css` 有這樣的 RWD 設定：
```css
.hero-container {
  display: flex;
  flex-direction: row; /* 桌機預設：左到右橫向排列 */
}

@media (max-width: 900px) {
  .hero-container {
    flex-direction: column-reverse; /* 手機轉換：改成上下垂直堆疊！ */
  }
}
```
僅僅只改變一個屬性，畫面立刻從並排變成上下排列的瀑布流，確保每個字體都在最舒適的閱讀大小。

---

## 3. 利用相對單位 (如 rem, vh, %) 取代寫死的 px
為了實現響應式，RWD 設計通常不喜歡把大小用絕對單位 `px`（像素）釘死。
例如我們設定高度用的是 `min-height: 75vh`（代表視窗總高度的 75%）而不是 `600px`；或者是寬度用 `100%`。
這種相對單位的設定方法，能讓網頁中的元素像「液體」一樣，倒進多大容量的杯子（手機、平板）都能恰到好處地延伸填滿！
